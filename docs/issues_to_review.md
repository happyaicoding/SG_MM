## Issue #006 — Web UI 外網存取方案

- **狀態**：⚪ 未啟動
- **現狀**：Phase 4 開始實作 Web UI 時再決定
- **候選方案**：
  1. **Cloudflare Tunnel + 自有網域**（推薦，年費 NT$300-500）
  2. **Port Forward + DDNS + Caddy + Let's Encrypt**（需處理憑證與認證）

- **預設值**：若 Phase 4 仍未決定，採用方案 1（Cloudflare Tunnel）
- **預計檢視時間**：Phase 4 開始時

## Issue 處理流程

### 開新 Issue
```
1. 確認該議題符合「Issue」定義（不是當下要做的，是後續要評估的）
2. 在本文件加入新條目，編號遞增
3. 在 CLAUDE.md 的 Phase Status 區塊提及（若會影響當前 Phase）
4. 寫進相關 phase_N_report.md（若是 Phase 結束時新發現）
```

### 觀察 Issue
```
每 Phase 結束時：
1. Claude Code 跑 phases_plan.md 規定的 Phase Gate 評估 script
2. 把該 Phase 相關的 Issues 狀態更新（🟡 → 🟢/🟠/🔴）
3. 異常情況提示使用者人工介入
```

### 解除 Issue
```
觀察結果符合預期：
1. 標記為 🟢 已解除
2. 寫入解除理由與當時數據
3. 保留條目（歷史紀錄）
```

### 升級 Issue
```
觀察結果需要修改規格：
1. 標記為 🔴 已升級
2. 在 docs/spec.md 對應章節更新
3. 在 CLAUDE.md 對應規則更新
4. 寫進當下 phase_N_report.md 的「設計決策變更」段落
```
